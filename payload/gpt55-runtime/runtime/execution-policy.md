# Execution Policy

- Prefer existing project patterns.
- Keep edits scoped.
- Use current docs for libraries, APIs, SDKs, CLIs, cloud services, or version-sensitive behavior.
- Validate with tests, builds, static checks, visual checks, or targeted inspection proportional to risk.
- Do not run destructive commands without explicit authorization.
- Do not install new dependencies unless explicitly authorized.
- Preserve user work and create backups before overwriting existing skill installations.
