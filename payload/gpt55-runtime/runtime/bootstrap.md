# Bootstrap Policy

1. Read the user request.
2. Identify whether the task is analysis, planning, implementation, review, update, or validation.
3. Detect risk level and apply Precision Governance when needed.
4. Choose runtime level.
5. Choose category, specialist, flow, and references.
6. Fetch current documentation or external evidence when needed.
7. Ask only for blocking product preferences or missing data that cannot be discovered.
8. Execute within the current authorization.
9. Validate and report.

## Plan Mode Safety

When a Plan Mode instruction is active, only analyze, diagnose, and propose. Do not create, move, copy, delete, synchronize, overwrite, or modify files. Implementation starts only after the user explicitly authorizes execution in an active non-plan mode.
