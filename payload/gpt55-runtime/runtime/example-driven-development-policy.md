# Example-Driven Development Policy

Before creating non-trivial code, project structure, UI, API, DevOps, tests, AI agent, or integration from scratch, look for reliable examples.

Priority:

1. Official documentation.
2. Official sample.
3. Official repository.
4. Maintained boilerplate.
5. Recognized template.
6. Technical recipe.
7. Community snippet with review.

Do not copy blindly. Check license, maintenance, security, compatibility, and currentness.
