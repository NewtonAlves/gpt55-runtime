# Dev Examples Guide

Use this reference when creating non-trivial code, project structure, API, UI, app, game, AI agent, tests, or DevOps work.

Priority:

1. Official documentation.
2. Official sample.
3. Official repository.
4. Maintained boilerplate.
5. Recognized template.
6. Technical recipe.
7. Community snippet after review.

Checklist:

- license acceptable;
- maintained recently enough for the stack;
- compatible versions;
- no unsafe scripts;
- adapted to local project style;
- validated after adaptation.
