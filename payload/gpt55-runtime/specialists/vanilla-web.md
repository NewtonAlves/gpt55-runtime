# Vanilla Web Specialist

Use for HTML, CSS and JavaScript without React, Next, Vue or another frontend framework, especially when the user wants lightweight pages, forms, UI interactions, dashboards, CRUD interfaces or PHP frontend integration.

Prefer:

- MDN;
- web.dev Forms;
- W3C WAI;
- semantic HTML;
- accessible forms;
- mobile-first CSS;
- visible focus states;
- `addEventListener`;
- Fetch API.

Avoid:

- inline event handlers;
- placeholder-only labels;
- inaccessible forms;
- `innerHTML` with untrusted data;
- localStorage for sensitive data;
- unnecessary framework assumptions.

Fallback: `frontend` plus `critic-reviewer`.

Related flows: `php-fullstack-flow`, `design-system-flow`.
