# Frontend Category

Use for web UI, React, Next.js, dashboards, landing pages, design implementation, accessibility, and browser validation.

Do not use for backend-only work unless UI behavior is affected.

Aliases: web, ui, react, next, dashboard, landing, frontend.

Specialists: `react`, `nextjs`, `critic-reviewer`.

Flows: `saas-flow`, `design-system-flow`, `example-driven-development-flow`.

References: UI references, component patterns, design prompts, dev examples.

Cost: medium by default, high for visual redesigns.

Validation: run relevant tests/builds and visually inspect important UI states when possible.

Precision: fetch current framework docs when API or version behavior matters.
