# Backend Category

Use for APIs, services, databases, ORMs, auth integration, jobs, queues, data contracts, and server behavior.

Do not use for purely visual UI work.

Aliases: api, server, database, orm, prisma, backend.

Specialists: `prisma`, `openai`, `mcp`, `critic-reviewer`.

Flows: `backend-api-flow`, `auth-flow`, `example-driven-development-flow`.

References: backend, dev examples, source verification.

Cost: medium; high for migrations, auth, production, or data loss risk.

Validation: tests, schema checks, build, smoke path, and security review when auth/data are involved.
