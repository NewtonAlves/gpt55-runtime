# Security Category

Use for auth, permissions, secrets, threat modeling, vulnerability review, dependency risk, production access, and sensitive data.

Aliases: seguranca, auth, permissao, token, segredo, vulnerabilidade.

Specialists: `critic-reviewer`, `evidence-reviewer`, `mcp`.

Flows: `auth-flow`, `backend-api-flow`, `multi-agent-review-flow`.

Cost: high by default.

Validation: inspect actual code/config and avoid destructive or credential-touching actions without explicit authorization.
